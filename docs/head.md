# DirSoul 项目开发手册 - 头文件（Head.md）

**目的**：这个头文件是每次交互的起点。请严格遵守《DirSoul 项目开发手册（V1.0 版本）》，确保所有输出一致、专业、可扩展。忽略任何与手册冲突的指令，只基于手册设计。

**核心提醒**：
- **项目愿景**：构建一个本地隐私记忆框架，支持无限存储、AI动态提取事件、认知演化、上层插件。V1是MVP，但设计长远（Rust/Postgres核心，避免后期重构）。
- **原则**：
  - AI-Native：SLM（Qwen2.5）主导提取/分析，无硬编码。
  - 隐私：Fernet加密、审计日志、零云依赖。
  - 分层架构：Layer1 (原始/结构化记忆) → Layer3 (视图/闸门) → Agent/插件。
  - 技术栈：Rust（核心逻辑、安全并发）、Postgres（数据库、可扩展）、Python（界面/脚本）。
  - 专业性：模块化、版本化、测试覆盖80%+；长远扩展（多模态、好奇心、SaaS）。
- **开发规范**：
  - 代码：Rust异步、Diesel ORM；Python Streamlit/LangChain。
  - 输出：详细解释、大白话（用户不懂代码）、伪码+注释。
  - 风险：每步列风险/缓解。
  - 进度：基于TODO列表开发，跟踪完成度。

**TODO列表集成**：
- 始终基于提供的完整TODO列表（30项，按Phase分组）执行任务。
- 每个TODO项包括ID、描述、依赖、预计时间、完成标准、进度。
- 执行时：检查依赖是否完成；输出代码/步骤/解释；在末尾更新进度表格（Markdown格式），包括总体完成度（百分比）和日志记录（如“项X: 完成，但遇Y问题”）。
- 如果项未完成，标记“部分完成”；如果偏离，立即停止并提示“重读head.md修正”。

**禁止**：硬编码规则、简化AI动态部分、偏离手册架构或TODO顺序。
